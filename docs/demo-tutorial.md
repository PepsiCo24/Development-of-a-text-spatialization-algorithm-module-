# 基于填图对象智能识别的文本空间化算法模块

## 系统演示操作指南

版本：v1.1（优化演示数据版）  
建议时长：15-20 分钟  
适用场景：项目验收、答辩汇报、专家评审、功能培训

> 本指南中的地质内容均为功能演示而构造，不代表真实勘查成果，不得用于生产决策。

## 1. 演示目标

完整展示从多格式地质资料入库，到解析/OCR、实体识别、属性关系抽取、术语标准化、空间化、知识图谱、证据问答、分析报告和成果导出的闭环。所有演示结果均应能回到来源页码和原文段落核查。

## 2. 演示资料包

| 文件 | 格式 | 主要用途 |
| --- | --- | --- |
| 01-铜绿山矿段综合地质调查报告 | TXT | 主流程；18个实体、7项属性、6类关系、4个点线面对象 |
| 02-ZK001钻孔地质编录表 | DOCX | Word在线预览、分层编录、厚度和品位解析 |
| 03-铜绿山矿段矿化特征简报 | PDF | 可检索PDF解析、章节与段落证据 |
| 04-F1断裂野外记录扫描件 | 扫描PDF | PaddleOCR、坐标、走向、倾向和倾角 |

主资料中的稳定预期结果：

- 18个已确认实体，包含地层、岩性、岩体、断裂、矿体、矿种、矿化带、年代、地名、坐标、品位、厚度、倾向和倾角。
- 7项属性，覆盖年代、岩性、厚度、规模、品位、倾向和倾角。
- 6类关系：位于、赋存于、侵入、接触、控制、包含。
- 4个空间对象：ZK001钻孔点、Ⅰ号铜铁矿体点、F1断裂线、铜绿山调查区域面。

## 3. 演示前检查

### 3.1 服务与账号

1. 运行 `.tools/start-all.ps1`，等待所有服务启动。
2. 浏览器访问 `http://127.0.0.1:5173`。
3. 管理员账号：`admin / admin123`。
4. 普通用户账号：`zky / user123`（如密码已调整，以系统管理员设置为准）。
5. 工作台确认 Spring Boot、FastAPI、PostgreSQL 均显示“正常”。

### 3.2 模型与OCR

1. 使用管理员账号进入“系统管理 - 模型配置”。
2. 确认 Qwen 的地址为 `https://api.siliconflow.cn/v1`，模型名称和密钥已保存并启用。
3. 不在投影屏幕上展示或朗读完整API Key。
4. 首次OCR前确认 PaddleOCR/PaddlePaddle 已安装；扫描PDF第一次解析可能因模型加载而稍慢。

### 3.3 建议现场策略

先用“恢复标准演示数据”展示确定性完整结果，再上传 Word、PDF 和扫描 PDF 展示多格式处理。大模型调用受网络影响，正式答辩不要从零开始等待所有模型任务。

## 5. 推荐演示流程

### 步骤1：登录与权限区分（约1分钟）

操作：使用管理员登录，指向右上角“系统管理员/管理员”标识；打开系统管理，展示用户管理。可补充说明普通用户登录后显示“普通用户”，且不可进入管理员专属功能。

验收点：角色标识清晰；管理员可管理用户和系统配置；普通用户没有管理员权限。

### 步骤2：基础数据资源池（约1分钟）

操作：进入“资料资源池”，使用名称、区域、类型和状态筛选；打开主资料详情；展示名称、类型、区域、年份、关键词、摘要和处理状态。

验收点：支持PDF、DOCX、TXT和常见图片格式；支持查询、筛选、预览、编辑和删除；旧版`.doc`会明确提示另存为`.docx`。

### 步骤3：文档解析、快速录入与OCR（约2分钟）

操作A：打开标准演示资料的“智能解析”，展示5个章节文本块、页码和处理进度。  
操作B：在资料资源池使用“粘贴文本演示”，加载预置文本并创建任务。  
操作C：上传`04-F1断裂野外记录扫描件.pdf`，启动解析，展示OCR结果。

扫描件预期识别关键词：`F1断裂`、`114.9310°E`、`30.0790°N`、`北东`、`南东`、`68°`。

验收点：原件和解析文本可切换；异常信息明确；扫描PDF通过PaddleOCR产生文本，而不是返回空内容。

### 步骤4：地质实体识别与人工审核（约2分钟）

操作：进入“实体识别”，选择标准演示资料；按实体类型筛选；点击原文高亮与右侧实体卡片；展示置信度、来源页、证据、模型和审核状态。

重点实体：`下三叠统大冶组`、`中厚层状灰岩`、`闪长玢岩`、`F1断裂`、`Ⅰ号铜铁矿体`、`黄铜矿`、`磁铁矿`、`25.60 m`、`1.26%`、`38.40%`、`南东`、`68°`。

人工审核：演示新增实体、修改名称或类型、确认结果；说明管理员和普通用户均可在授权范围内维护业务识别结果，不再出现403。

验收点：不同实体显示不同的实际置信度；置信度按百分比正确显示；新增、修改、删除、确认均成功。

### 步骤5：属性、关系与术语标准化（约2分钟）

操作：进入“知识抽取”，依次打开属性、关系和标准化页签。

属性预期：

1. 大冶组 - 年代 - 早三叠世。
2. 大冶组 - 岩性 - 中厚层状灰岩。
3. Ⅰ号铜铁矿体 - 厚度 - 25.60 m。
4. Ⅰ号铜铁矿体 - 规模 - 走向延伸约420 m。
5. Ⅰ号铜铁矿体 - 品位 - Cu 1.26%、Fe 38.40%。

六类关系预期：

1. Ⅰ号铜铁矿体 位于 铜绿山矿段。
2. Ⅰ号铜铁矿体 赋存于 矽卡岩化带。
3. 闪长玢岩 侵入 下三叠统大冶组。
4. 闪长玢岩 接触 矽卡岩化带。
5. F1断裂 控制 Ⅰ号铜铁矿体。
6. Ⅰ号铜铁矿体 包含 黄铜矿。

人工审核：分别演示新增属性、修改关系、确认和删除测试项，再恢复正确内容。

验收点：属性和关系支持完整增删改查及人工确认；标准化同时显示原始名称、标准名称和匹配状态。

### 步骤6：文本空间化与地图联动（约2分钟）

操作：进入“空间地图”，选择标准演示资料，点击“全图”。依次开启/关闭钻孔、矿点、断裂和调查区域图层；切换标准地图与卫星影像；移动鼠标展示坐标；使用距离和面积量算。

预期对象：

- ZK001钻孔：Point `[114.9384, 30.0840]`。
- Ⅰ号铜铁矿体：Point `[114.9392, 30.0836]`，标注为钻孔约束推定。
- F1断裂：LineString，起点`[114.9310, 30.0790]`，终点`[114.9470, 30.0890]`。
- 铜绿山调查区域：Polygon，由4个角点闭合。

点击地图对象后，查看对象名称、类型、坐标、资料、页码和原文；再点击“查看原文”，确认页面定位到对应章节并高亮证据。

验收点：点、线、面位置与原文坐标一致；对象与原文可以双向定位；面积与距离量算有可见结果。

### 步骤7：知识图谱、关系展开和最短路径（约2分钟）

操作：进入“图谱问答”，等待图谱状态完成。查询`F1断裂`并展开一层关系；再选择`F1断裂`为起点、`黄铜矿`为终点，点击“查找最短路径”。

预期路径：`F1断裂 --控制--> Ⅰ号铜铁矿体 --包含--> 黄铜矿`。图中应突出路径节点和连线，结果区显示关系文字。点击矿体节点可联动空间位置。

验收点：节点查询、关系展开、最短路径均有可见反馈；无路径时明确提示，而不是点击无反应。

### 步骤8：证据化智能问答（约2分钟）

推荐问题及答案要点：

1. `F1断裂如何控制Ⅰ号铜铁矿体？` - 返回控制关系、断裂空间位置、来源第3页。
2. `Ⅰ号铜铁矿体的厚度、规模和品位是多少？` - 25.60 m、约420 m、Cu 1.26%、Fe 38.40%，来源第5页。
3. `ZK001钻孔的孔口坐标和揭矿区间是什么？` - 114.9384°E、30.0840°N，188.70-214.30 m，来源第4页。
4. `闪长玢岩与大冶组和矽卡岩化带是什么关系？` - 侵入大冶组、接触矽卡岩化带，来源第2页。

验收点：回答显示相关实体、空间位置、来源资料、页码和来源段落；“查看依据”可回到原文。

### 步骤9：分析报告、成果导出与系统治理（约2分钟）

操作：进入“系统管理 - 成果管理”，选择标准演示资料；生成并预览空间化分析报告；导出PDF。再分别选择Excel、CSV、JSON和GeoJSON导出。

说明：GeoJSON可直接导入QGIS；“一张图推送”会把空间成果发送到已配置的汇聚接口。展示任务管理、模型调用日志、执行状态和异常信息。

验收点：报告可生成、预览和导出；四种结构化格式均可下载；推送入口、恢复标准演示数据入口、用户和日志管理均可见。

## 7. 常见故障处理

| 现象 | 原因与处理 |
| --- | --- |
| 401 | 登录过期；退出后重新登录，检查前端是否携带Bearer Token |
| 403 | 权限不足；确认右上角角色，管理员操作需使用admin账号 |
| AI服务503 | 检查8000端口、模型地址、模型名称和Key；硅基流动地址使用`https://api.siliconflow.cn/v1` |
| OCR无结果 | 检查PaddleOCR/PaddlePaddle、首次模型下载和`OCR_LANGUAGE=ch` |
| 地图点偏移 | 确认原文坐标系为WGS84；推定点与原文点应区分显示 |
| 图谱最短路径无反应 | 先构建图谱，确保起终点均已选择；无路径时查看提示 |
| Word无法预览 | 仅支持DOCX；旧DOC需在Word中另存为DOCX |
