# GeoText 演示资料与使用教程

用于逐项讲解功能的现场话术见 [项目功能演示讲稿](project-demo-speech.md)。

本教程用于现场展示 GeoText 从地质原文到可核查空间知识的完整流程。所有示例资料均为项目内构造的演示样本，仅用于功能验证与演示，不代表正式勘查结论。

## 1. 演示资料包

资料位于 [`demo-data`](../demo-data/)，均为 UTF-8 文本，可在页面上传或批量导入。

| 资料 | 推荐展示点 | 原文中的关键证据 |
| --- | --- | --- |
| `01-鄂东南铁铜矿调查.txt` | 区域范围、多边形、主断裂、花岗岩—灰岩接触带和铁铜矿化 | 4 个范围控制点；北东向主断裂；花岗岩侵入寒武系灰岩 |
| `02-宜昌磷矿地层调查.txt` | 地层关系、矿层属性、术语标准化 | 陡山沱组、寒武系水井沱组；厚度 3.6 米；P2O5 24.8% |
| `03-大冶矿区钻孔记录.txt` | 钻孔坐标、矿体属性、空间点对象 | ZK12；114.9380°E / 30.1120°N；埋深 212—239 米；TFe 46.2% |

## 2. 演示前准备

1. 启动系统后打开 [http://127.0.0.1:5173](http://127.0.0.1:5173)，使用 `admin / admin123` 登录。
2. 在“系统管理 → 模型配置”中选择 DeepSeek 或 Qwen，填写对应 API Key，点击“保存并应用”。确认模型卡片显示“KEY 已配置”。
3. 推荐使用 DeepSeek 或 Qwen 中一个服务贯穿全程；切换模型时可重复执行识别或抽取，以展示不同模型配置的可控性。

> 没有模型 API Key 时，仍可完成资料上传、文档解析、资料预览和系统管理演示；实体识别、知识抽取、空间化、图谱同步和证据问答需要已配置的模型服务。

## 3. 导入资料

### 页面导入

1. 进入“资料资源池”，点击“上传资料”。
2. 依次选择三份 `.txt` 文件，建议填写下列元数据：

| 文件 | 区域 | 年份 | 关键词 |
| --- | --- | --- | --- |
| 鄂东南铁铜矿调查 | 鄂东南 | 2024 | 铁铜矿、断裂、花岗岩 |
| 宜昌磷矿地层调查 | 宜昌 | 2024 | 磷矿、震旦系、寒武系 |
| 大冶矿区钻孔记录 | 大冶 | 2025 | 钻孔、铁矿体、品位 |

3. 上传完成后，资料状态应为“待解析”。

### 一键导入

在项目根目录运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\import-demo-data.ps1 -BaseUrl http://127.0.0.1:8080
```

脚本使用演示管理员账号登录并上传三份资料；重复执行会再次新增资料，因此现场演示时只执行一次。

## 4. 推荐的 12 分钟演示流程

### 第 1 分钟：资料可追溯入库

打开“资料资源池”，展示名称、区域、年份、关键词和待解析状态。点击任一资料的“预览”，说明原始资料会被保留，后续所有结果均可回到来源文本。

### 第 2—3 分钟：智能解析

1. 选择 `01-鄂东南铁铜矿调查`，点击“智能解析”。
2. 在右上角点击“开始解析”。
3. 完成后展示右侧的分块文本和章节/页码信息。

建议话术：

> 系统先把原始资料拆成带页码的结构化文本块，后面的实体、关系、坐标和问答答案都能回到这里核查。

对三份资料均执行解析。TXT 资料通常很快完成；实际耗时取决于机器和文件大小。

### 第 4—5 分钟：实体识别

1. 进入“实体识别”，选择已解析的 `01-鄂东南铁铜矿调查`。
2. 选择已配置的模型，点击“开始识别”。
3. 完成后点击左侧原文中的彩色实体，展示右侧的实体类别、置信度、来源页码、模型名称和原文证据。

重点观察：`寒武系灰岩`、`奥陶系碳酸盐岩`、`燕山期花岗岩体`、`北东向主断裂`、`铁铜矿化带`。

随后对 `03-大冶矿区钻孔记录` 执行识别，重点展示 `ZK12`、`一号铁矿体`、`闪长岩`、`大冶组灰岩`。

### 第 6—7 分钟：属性、关系与术语

1. 进入“属性、关系与术语”，选择已完成实体识别的资料。
2. 点击“开始抽取”。
3. 依次查看“关系网络”“实体属性”“术语标准化”三个标签。

推荐展示 `02-宜昌磷矿地层调查`：

- 关系：陡山沱组包含主要磷矿层；北西向断裂切割矿层。
- 属性：矿层平均厚度 3.6 米、P2O5 平均品位 24.8%。
- 标准化：地层、矿体和构造名称会保留原称与标准名称。

### 第 8—9 分钟：文本空间化地图

1. 进入“空间地图”，选择 `01-鄂东南铁铜矿调查`。
2. 点击“开始空间化”。
3. 完成后切换“标准地图 / 卫星影像”，并点击图层开关和“全图”。
4. 在证据轨道中点击对象，说明地图对象、中心坐标、来源页码和原文会同步联动。

推荐重点：区域范围应呈面对象，主断裂应呈线对象，中心坐标和矿化相关对象应呈点对象或带有空间证据的对象。

再切换到 `03-大冶矿区钻孔记录`，展示 ZK12 坐标对应的钻孔点。

### 第 10—11 分钟：图谱与证据问答

1. 进入“图谱问答”，选择已完成空间化的资料，点击“构建图谱与索引”。
2. 完成后点击图中的节点，或在“节点查询”中输入 `主断裂`、`ZK12`、`陡山沱组`。
3. 在“地质智能问答”中依次使用下列问题：

| 问题 | 预期展示 |
| --- | --- |
| `鄂东南调查区的主要控矿构造是什么？它控制何种矿化？` | 主断裂与铁铜矿化带之间的关系，以及来源段落 |
| `ZK12 钻孔揭露的主铁矿体埋深、厚度和平均品位是多少？` | 212—239 米、27 米、TFe 46.2%，并显示来源证据 |
| `宜昌磷矿层赋存于哪个地层？平均厚度和 P2O5 品位是多少？` | 陡山沱组及 3.6 米、24.8% 等属性证据 |

强调回答右侧的“来源段落”：系统并非只输出结论，而是同时给出可回查原文。

### 第 12 分钟：成果交付与治理

进入“系统管理”：

1. “成果导出”选择资料后，分别展示 XLSX、CSV、JSON、GeoJSON。
2. XLSX 用于四类成果汇总；GeoJSON 可直接导入 GIS 软件。
3. “任务管理”展示解析、实体、知识、空间、图谱的阶段状态。
4. “调用日志”展示模型、功能、状态、耗时和异常记录。

## 5. 现场演示顺序与常见处理

1. 先演示 `01-鄂东南铁铜矿调查`，它同时包含区域面、断裂线和矿化关系，视觉效果最好。
2. 用 `03-大冶矿区钻孔记录` 补充精确坐标、钻孔和数值属性。
3. 用 `02-宜昌磷矿地层调查` 补充地层—矿层关系与品位问答。
4. 识别或抽取失败时，先检查“系统管理 → 模型配置”是否显示“KEY 已配置”，再在任务页使用“重试”。
5. 首次构建图谱时，BGE-M3 模型和 Qdrant 索引可能需要额外时间；现场应优先完成解析、实体、知识和空间化，再进行图谱同步。

## 6. 演示结束核查清单

- 三份资料均可在资源池检索到。
- 至少一份资料显示“已解析”“识别完成”“抽取完成”“空间化完成”“图谱同步完成”。
- 地图中可点选空间对象并看到来源证据。
- 图谱中存在节点和关系，问答结果包含来源段落。
- 已下载至少一个 XLSX 或 GeoJSON 成果包。
