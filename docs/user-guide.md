# 系统使用指南

## 1. 登录与角色

访问 `http://127.0.0.1:5173`。初始管理员为 `admin / admin123`；管理员可在“系统管理—用户管理”新建普通用户或管理员。普通用户可完成资料处理和成果浏览，管理员另可配置模型、管理账号、恢复演示数据和查看日志。

## 2. 从文本到空间成果

1. 在“资料资源池”上传 PDF、Word、TXT 或图片，补充区域、年份、关键词和摘要；也可点“粘贴文本演示”，装入预设钻孔记录或粘贴任意地质文本。
2. 打开资料并执行“智能解析”。页面同时保留原始资料、解析后的章节文本、进度和失败原因。
3. 在“实体识别”选择 DeepSeek 或 Qwen，开始识别。可按实体类型、置信度、待核/确认/驳回状态过滤；点击原文高亮可查看页码、模型和证据。
4. 选择实体后，可通过“补充实体、修改、确认、驳回、删除”完成人工校核。实体、属性和关系接口均保留相同的人工修订能力。
5. 在“属性、关系与术语”执行知识抽取，检查年代、厚度、品位、规模、岩性，以及位于、赋存于、侵入、接触、控制、包含等关系；在“术语标准化”核对原名、标准名和别名匹配结果。
6. 在“空间地图”生成地名、坐标、矿点、钻孔、断裂和调查区域的点线面成果。使用图层、底图、量算、图例和对象证据栏进行核查。
7. 在“图谱问答”同步图谱与向量索引，然后进行节点查询、关系展开、路径查询和带来源段落的自然语言问答。
8. 在“系统管理—成果导出”导出 Excel、CSV、JSON 或 GeoJSON。管理员可调用 `POST /api/admin/one-map/push?documentId={id}` 将 GeoJSON 交付给本地一张图原型；设置 `ONE_MAP_WEBHOOK_URL` 后会推送至外部汇聚接口。

## 3. 演示数据恢复与模型配置

- 管理员调用 `POST /api/admin/demo-data/restore` 可恢复“大冶矿区钻孔记录（标准演示）”，系统会自动启动解析。
- 在“系统管理—模型配置”填写 DeepSeek 或 Qwen 的 API 地址、模型、Key、temperature 和提示词，然后保存并应用。未配置 Key 时，上传、解析和资料管理仍可演示；智能识别、知识抽取、空间化和问答需要模型服务可用。

## 4. 常见问题

- 解析失败：检查资料格式、服务 `http://127.0.0.1:8000/docs` 是否可访问，并在解析页点“重试解析”。
- LLM 失败：检查管理员配置的 Key 与模型地址，并在任务页重试。
- 地图无对象：先完成实体和知识抽取，再执行空间化；文本中应包含明确的坐标、地名或范围描述。
- 数据库升级：执行 `powershell -NoProfile -ExecutionPolicy Bypass -File .\.tools\migrate-db.ps1`。
